### 3. Create Embedding Service

**File:** `src/lib/services/embedding-service.ts`

```typescript
// Env vars: OLLAMA_BASE_URL (default: http://192.168.178.50:11434)
// Model: qwen3-embedding:8b
// Output dimensions: 4096

// embed(text: string) → Promise<number[]>
//   - POST to ${OLLAMA_BASE_URL}/api/embed
//   - Body: { model: 'qwen3-embedding:8b', input: text }
//   - Parse response: response.embeddings[0] (Ollama returns array of embeddings)
//   - Validate: result is array of 4096 numbers
//   - Retry logic: 3 retries with exponential backoff (1s, 2s, 4s)
//     - Ollama may need time to load the model on first request
//   - Throw clear error if all retries fail

// embedBatch(texts: string[]) → Promise<number[][]>
//   - POST to ${OLLAMA_BASE_URL}/api/embed
//   - Body: { model: 'qwen3-embedding:8b', input: texts }
//   - Ollama supports batch input — returns embeddings array matching input order
//   - If batch too large (>10 texts), split into chunks of 10 and process sequentially
//   - Return array of vectors in same order as input
```

### 4. Create Chunking Service

**File:** `src/lib/services/chunking-service.ts`

```typescript
// Types:
// Chunk { text: string, index: number, startChar: number, endChar: number }
// DocumentMetadata { documentId: string, name: string, sourceType: string, crmCustomerId?: string, userId: string }
// PreparedChunk { text: string, index: number, metadata: DocumentMetadata }

// chunkText(text: string, options?: { chunkSize?: number, overlap?: number }) → Chunk[]
//   - Default chunkSize: 2000 characters
//   - Default overlap: 300 characters
//   - Algorithm:
//     1. Split text into paragraphs (split on \n\n)
//     2. Accumulate paragraphs until approaching chunkSize
//     3. When chunk is full, start new chunk with overlap from end of previous
//     4. If a single paragraph exceeds chunkSize, split at sentence boundaries
//     5. If a single sentence exceeds chunkSize, split at chunkSize (hard break)
//   - Return array of Chunk objects with text, index, and character positions

// chunkDocument(content: string, metadata: DocumentMetadata) → PreparedChunk[]
//   - Call chunkText to get chunks
//   - Attach metadata to each chunk
//   - Return PreparedChunk array ready for embedding + storage
```

### 5. Create Documents Table Migration

**File:** `supabase/migrations/<next_number>_create_documents.sql`

Determine the next migration number by checking existing files in `supabase/migrations/`.

```sql
CREATE TABLE documents (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  crm_customer_id uuid REFERENCES crm_customers(id) ON DELETE SET NULL,
  name text NOT NULL,
  source_type text NOT NULL,        -- 'asset', 'meeting', 'drive_file', 'note', 'upload'
  source_id uuid,                   -- FK to the source record (polymorphic, no constraint)
  chunk_count integer DEFAULT 0,
  embedded_at timestamptz,          -- NULL = pending embedding
  metadata jsonb DEFAULT '{}',
  created_at timestamptz DEFAULT now()
);

-- Indexes
CREATE INDEX idx_documents_user ON documents(user_id);
CREATE INDEX idx_documents_customer ON documents(crm_customer_id);
CREATE INDEX idx_documents_pending ON documents(user_id) WHERE embedded_at IS NULL;
CREATE INDEX idx_documents_source ON documents(source_type, source_id);

-- RLS: users see own documents only
ALTER TABLE documents ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can view own documents"
  ON documents FOR SELECT
  USING (auth.uid() = user_id);

CREATE POLICY "Users can insert own documents"
  ON documents FOR INSERT
  WITH CHECK (auth.uid() = user_id);

CREATE POLICY "Users can update own documents"
  ON documents FOR UPDATE
  USING (auth.uid() = user_id);

CREATE POLICY "Users can delete own documents"
  ON documents FOR DELETE
  USING (auth.uid() = user_id);
```

**File:** `src/app/api/documents/upload/route.ts`

```typescript
// POST with multipart form data:
//   - file (File) OR text (string) — one of these is required
//   - name (string) — document name
//   - crm_customer_id (string, optional) — link to CRM customer
//   - source_type (string, default: 'upload')
//   - source_id (string, optional)
//
// Processing:
// 1. Verify authenticated user
// 2. Extract text content:
//    - If text provided: use directly
//    - If file provided: read file as text (plain text, markdown, etc.)
//    - For binary files (PDF, DOCX): return 400 with message "Binary file support coming soon" (handled in Prompt 24)
// 3. Create document record in Supabase via admin client:
//    { user_id, name, source_type, source_id, crm_customer_id, metadata: { original_size: text.length } }
// 4. Chunk the content using chunking-service
// 5. Embed all chunks using embedding-service (embedBatch)
// 6. Ensure Qdrant collection exists (ensureCollection)
// 7. Upsert points to Qdrant:
//    - Point ID: use UUIDs (generate with crypto.randomUUID())
//    - Vector: the embedding
//    - Payload: { document_id, chunk_index, content, source_type, crm_customer_id, user_id, name }
// 8. Update document record: chunk_count = N, embedded_at = now()
// 9. Return: { document_id, chunk_count, name }
// 10. Handle errors: if embedding fails, leave embedded_at as null (will be retried by cron in Prompt 24)
```
